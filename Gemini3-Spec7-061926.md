## 系統核心願景 (Core Vision)
AegisReview Pro 是一款專為中華民國衛生福利部食品藥物管理署 (TFDA) 醫療器材審查人員打造的「全代理人導向智慧查驗登記審查系統（Agentic AI Reviewing System）」。本系統旨在協助審查官解決高技術密度、高合規複雜度的醫療器材技術檔案（STD）審查痛點。
透過異質文件自動化解析、雙向 Markdown-AST 同步編輯、動態特殊協作協議演化進化（Dynamic Skill Mutation）以及具備 D3 與 Recharts 強大底層的「WOW 互動式大師儀表板」，本系統可將冗長、複雜的工程與物理學數據轉化為安全、可追溯且高可讀性的結構化產出。本說明書將以 4,500 字以上之深度技術純度，定義其完整的軟體工程規格與實踐範式。

## 第一章：兩階段智慧審查代理人流水線與異質文檔抽取
本系統採用高防禦、高精度、高容錯的雙階段 AI 審查設計流程，將原本分散雜亂的廠商申報材料進行重構，其核心架構圖如下：

```
上傳申報案卷 (PDF/TXT/MD)
      |
      v
[第一階段：高容錯文本層析引擎] ---> 語意分群(Chunking) & 混合檢索(RAG+Search Grounding) -> 主動模型推理
      |
      v
[生成：3000~4000字 階段一技術摘要] (包含 5 大核心分析表格，支援 AST 雙向 markdown 雙編輯器視圖)
      |
      +-----------------------------------------+
                                                |
                                                v
[第二階段：自適應協議變異演化] <--- 載入 default skill.md (TFDA-MD-AI-Protocol-5000)
      |
      v
[生成：客製特化 dynamic_specialized_skill.md] (開放官員手動修正、下載，特異化本案小兒/聯網電磁等規約)
      |
      v
[最終生成：3000~4000字 綜合審查評估總結] (包含至少 20 個技術實體上下文對位註解大表 + 20 個跟進問題)
```

### 1.1 高精度非結構化 RAG 抽取與混合 Grounding 引擎
1. **多重 Layout 解析與行列表格定位器 (Table Extractor)：** 廠商提報的報告往往包含雜亂、未光學對位或轉檔跑版的表格。本模組在 Node.js 中使用基於 PDF.js 的物理定位解析引擎，將每個文字單元的 `(X, Y)` 座標、字體粗細與字距進行二維矩陣映射。當行與列的物理重疊率達到 85% 以上時，自動將資料推入表格還原管線，轉化為標準的 Markdown Table 格式，避免傳統 LLM 逐字元閱讀（Sequential Token Reading）時引起的行列欄位混亂錯位。
2. **混合即時網頁 Grounding 機制：** 對接 Gemini API 的 `GoogleGenAI` 工具，並在調用 `google.genai.models.generateContent` 時啟用 `googleSearch` 工具。此技術使得 AI 代理人在遭遇罕見的技術詞彙（如特定的高溫射頻燒灼阻抗數值）或新釋出的 TFDA 法規草案時，可以主動發起並行網頁抓取，獲取台灣法規資料庫的最即時（Latest Live）法理證據，並將 URL 回傳至審查報告中。

### 1.2 階段一技術摘要：3,000 ~ 4,000 字生成深度與 5 大核心分析表格
在第一階段，審查官可自由選擇模型（如具備高智商推理深度的 `gemini-2.5-pro`或極致效率 `gemini-2.5-flash`），調整 Prompt 側記參數，系統將自適應生成超長篇幅之「查驗登記初步審查摘要書」（字數嚴格控制在 3,000 至 4,000 字之間）。該摘要書內部必須硬性包含以下 5 大核心分析矩陣（表格），展現極高的技術精緻度：

#### 表格 1：行政合規完備性與字元級一致性交叉審查矩陣 (Administrative Registry)
* **目的：** 確保申報人、原廠授權書、自由銷售證明與製造廠符合性之合規性無任何字元級落差。
* **主要欄位與技術參數：**
  - `審查項目`：藥商名稱、製造廠名稱與地址、原廠 LOA 簽字效力、出產國 CFS 與駐外簽證字號、我國 QMS/GMP 符合性證書登錄號。
  - `查驗申報書記載`：例如 \"MedTech Innovations LLC\"
  - `第三方佐證文件記載`：例如 \"MedTech Innovations Limited Liability Company\"
  - `字元級相似度阻斷分析`：計算 Levenshtein 距離（相似度小於 95% 時拋出預警）。
  - `合規裁決與註解`：[WARNING] 英文名稱縮寫不一致，廠商需提具變更說明。

#### 表格 2：實質等同性 (Substantial Equivalence) 技術物理與能量特質對照表格 (SE Matrix)
* **目的：** 對照本受審器材與境內已核准上市對照品（設計之 Predicate Device）的所有工程參數。
* **主要欄位與技術參數：**
  - `比較大類`：主要物理結構、核心接觸塑料、驅動工作頻率、最大能量輸出、備用電源系統。
  - `本案受審器材 (Subject Device)`：規格描述（例如 AC $100\sim240\text{V}$, $50/60\text{Hz}$ / 內建 $11.1\text{V}$ 鋰離子電池）。
  - `境內已核准對照品 (Predicate Device)`：規格描述（例如 AC $110\text{V}$ $60\text{Hz}$ / 無電池備載）。
  - `差異深度安全性分析`：當引入內部鋰電池時，必須審查其是否通過 IEC 62133 安全測試。
  - `實質等同裁決`：[PASS - EQUIVALENT] 屬於技術效能等同且提升。

#### 表格 3：ISO 10993 材料生物學接觸與各項毒理試驗關鍵參數表 (Biocompatibility Deep-Dive)
* **目的：** 稽核與病患人體直接或間接接觸材質之生物相容性安全性數據。
* **主要欄位與技術參數：**
  - `試驗名稱`：細胞毒性 (ISO 10993-5)、致敏性反應 (ISO 10993-10)、原發性刺激反應 (ISO 10993-23)、中長期全身毒性及植入。
  - `接觸等級與部位`：例如「表面接觸 - 長期暴露（$24\text{小時}\sim 30\text{天}$）- 皮膚與粘膜接觸」。
  - `試驗最嚴苛前置處理條件`：洗提比例符合 $3\text{ cm}^2/\text{mL}$ / 老化後 / 極端浸提溫度 $37^\circ\text{C}$ / 處理 $24$ 小時。
  - `實驗室實測數值與百分比`：細胞溶解度評級實測值為 Grade 0 (細胞無裂解) / 天竺鼠過敏反應致敏率為 0%。
  - `符合性法理裁決`：[PASS] 完全符合本案適用之表面長期接觸安全測試標準。

#### 表格 4：IEC 60601-1 硬體電性與 IEC 60601-1-2 電磁相容電磁防禦測試極限參數表 (Electrical & EMC Safety)
* **目的：** 主動式儀器硬件與電磁環境耐受度查核。
* **主要欄位與技術參數：**
  - `法學與工程標準`：IEC 60601-1:2012 / IEC 60601-1-2:2014(Ed 4)。
  - `極端測試項目`：正常對地漏電流/單一故障對地漏電流；心電應用 CF 型耐壓保護；高壓靜電放電 (ESD) 空氣 $\pm15\text{kV}$ / 接觸 $\pm8\text{kV}$；$2.4\text{GHz}$ ISM 射頻高強度干擾輻射防禦。
  - `合規閾值極限 (Limit)`：患者漏電流最大應限制在 $\le 50\text{ }\mu\text{A}$ / ESD 信號不中斷 / 輻射干擾下維持基本性能（Essential Performance）。
  - `本案原始報告實測值`：對地漏電流正常狀態實測值為 $8.2\text{ }\mu\text{A}$ / 於 $2.4\text{GHz}$、強度 $28\text{ V/m}$ 連續干擾下，血氧偵測信號中斷長達 $8.0$ 秒。
  - `缺失點判定結論`：[CRITICAL DEFICIENCY] 數據中斷大於臨床安全規範 limit $2.5$ 秒，嚴重違反基本性能。

#### 表格 5：網路安全威脅建模漏洞掃描與軟體 SBOM 軟硬體材料防禦表 (Cybersecurity Vector)
* **目的：** 針對多參數生理監視器等通訊醫材，實施全面網路拓構與資安漏洞追蹤。
* **主要欄位與技術參數：**
  - `安全威脅來源`：空中下載 OTA 韌體更新遭篡改、低功耗藍牙 BLE 傳輸中間人竊聽 (MITM)、實體調試介面 JTAG/SWD 韌體代碼讀取、SBOM 第三方套件已知 CVE 高風險漏洞。
  - `現有安全控制強度`：簽章使用 RSA-3072 與 SHA-256 / BLE 採用「Just Works」配對 / eFuse 燒斷防止代碼倒退。
  - `加密強度與 key 等級推算`：128-bit AES 傳輸加密 / 未強制 LE Secure Connections。
  - `殘餘風險判定 (ALARP)`：[HIGH RISK] 「Just Works」配對極易遭中間人主動竊聽，無法達成合理最低風險 ALARP 等級。
  - `補件與整改工程指引`：限期廠商修改通訊晶片韌體，強制啟用 BLE Mode 1 Level 4，並使用數字對比（Numeric Comparison）進行配對。

### 1.3 頂級雙向 AST 同步 Markdown 編輯器
本系統主要編輯區域支援雙視窗設計：
- **Markdown 實時渲染視圖 (Live WYSIWYG Mode)：** 利用 `react-markdown` 進行無延遲排版渲染，配搭 Tailwind CSS 調和的精美邊框與排版樣式。
- **原始碼/Raw Text 編輯模式 (Markdown Code Mirror)：** 開放給資深審查官，可在無需離開 AI Studio iframe 的狀況下，極速輸入純文本或直接修改 Markdown 語意結構，兩邊在毫秒(ms)內通過 Abstract Syntax Tree (AST) 底層進行無失真、無遺漏的增量同步 (Incremental Bidirectional Synchronizer)。

## 第二章：階段二自適應技能演變與特化查驗報告
在 AegisReview Pro 中，審查邏輯並非一成不變，而是能在審查過程中「實現智能進化」。

### 2.1 協作協議自適應變異與進化架構 (Skill Mutation Framework)
系統提供預載之通用 `# AI 醫療器材技術審查協作協議 (TFDA 專用版 v3.0)`（即 `default skill.md`）。
1. **智能感應與特化：** 系統完成第一階段的 RAG 資料提取後，分析模型會自動嗅探到本案的「特異高風險因子」（例如：本監視系統包含針對『嬰幼兒』的心率算法、內建『藍牙無線遙測』以及『Tyvek 雙層高分子無菌密封包裝』）。
2. **重寫協作協議：** 系統呼叫進化代理人組件，將 `default skill.md` 的條文在記憶體緩衝區中進行增量編撰，為其量身定制一組針對「嬰幼兒生理限制防護」、「藍牙 Mode 1 Level 4 強制防禦」與「Arrhenius 加速老化天數二階公式稽查」的特異性審查條款。
3. **下載與重載機制：** 演化重寫後的協作協議被命名為 `dynamic_specialized_skill.md`。審查官可直接在界面上調看側邊欄變更對照圖，點擊手動微調，並提供 **[Download Specialized SKILL.md]** 實體檔案匯出按鈕，作為後續系列產品複查時的固定規則之用。

### 2.2 階段二綜合審查評估總結：3,000 ~ 4,000 字極緻產出

- **視覺呈現：** 雙 Y 軸、非線性流動回歸擬合雙階對數曲線。

- **關鍵特徵：** 藍色實線代表理論上隨著加速老化箱內溫度 ($25^\circ\text{C}$ 到 $60^\circ\text{C}$ 區間) 變化時，所需的「理論 Arrhenius 測試最少天數 ($AAT$)」。紅點代表「本案廠商實際執行的測試天數點位 (例：$55^\circ\text{C}$，120天)」。
- **操作互動：** 審查官可以直接用滑鼠在圖表上「拖拽溫度滑動桿（Slider）」，擬合曲線會動態更新（利用 $Q_{10}$ 模型即時重算），紅點若落入曲線下方安全性不夠之紅色不可靠陰影區，即刻代表測試不充足！。
- **資料來源：** Arrhenius 經典微分降解演算法動態求值。

### 5. 比對文件語意特徵高維投影散佈圖 (Semantic High-Dimensional Scatter Plot)
- **視覺呈現：** 基於 t-SNE / UMAP 二維度降維彩色散點分佈圖。
- **目的：** 透過向量相似度，尋找本次受審醫材的「預期用途與仿單」在 TFDA 上市 1,000 款類似品（歷史大數據）中的空間語意鄰近關係（等同性驗證）。
- **關鍵特徵：** 中心的密集彩色點群代表我國已批准上市、性能已被驗證可靠之標準監視器材。金黃色恆星點代表「本案受審器材」。
- **操作互動：** Hover 黃金點，系統連帶繪製出 3 條動態流星射線，指向與其距離最近的 3 個對照品（並閃爍顯示其核發登錄案號）。若黃金點點位落於邊緣極度孤立空白區，說明該預期用途極度偏離常規，應警示「不屬於實質等同」。
- **資料來源：** 送審材料與 TFDA 歷年核准仿單 Embedding 向量二維矩陣投影演算。

### 6. 軟體/電磁電控晶片多執行緒安全死鎖因果依賴網 (Entity Dependency Network)
- **視覺呈現：** 複雜的三維可收折力導向拓撲樹。
- **目的：** 投射本監視系統（C 級 major 高安全軟體等級）在多執行緒處理生理信號時的 Mutex 因果網絡。
- **關鍵特徵：** 網上 20～30 個核心節點代表：SpO2 信號讀寫、ECG 漏電保護、警報蜂鳴器線程、餵狗程序、藍牙封包佇列。當有節點出現紅色雙向箭頭閉環，說明該處極易在極端病患心率大於 300 bpm 發生 CPU「死鎖當機」，看門狗必須在 1.2 秒內強制 Reset。
- **操作互動：** 雙擊特定線程頂點，展開該程序在 SRS、SAD 架構說明書中的原始功能段落。
- **資料來源：** RAG 模型解析軟體 SRS（軟體需求說明）與架構手冊中所描繪的模組鏈拓構。
